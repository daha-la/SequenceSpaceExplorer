# docs/

Longer-form documentation that doesn't fit comfortably in a folder README —
currently just the visualizer's user guide, since the app has far more
panels, filters, and options than a `scripts/README.md` entry can cover
without drowning out everything else there.

| File | Contents |
|---|---|
| [`visualizer_guide.md`](visualizer_guide.md) | The SSE visualizer user guide: every panel and tool in the app (coordinates, filters, layers, colour, export, Boltz-2 structure prediction, RMSD), explained. |
| [`SSE_Visualizer_User_Guide.docx`](SSE_Visualizer_User_Guide.docx) | The same guide as a Word document, for anyone who'd rather read/annotate it outside a Markdown viewer (e.g. in a lab notebook or shared with a non-technical collaborator). |

The two are equivalent in content — `visualizer_guide.md` is the source of
truth; `SSE_Visualizer_User_Guide.docx` is generated from it. If you're
editing the guide, edit the Markdown and regenerate the Word document from
it, rather than editing the two independently.
