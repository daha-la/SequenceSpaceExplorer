Sequence Space Explorer · Pipeline walkthrough

Turning a pile of protein sequences into an explorable map

Every tool in this pipeline does one job: it reads the shared table of sequences, computes something, and appends new columns — never reordering or dropping rows. Follow the data from a raw sequence file all the way to an interactive landscape you can lasso, cluster, and fold.

input data read computation / transformation new columns written back

The thing that ties it all together

One typed table is the single source of truth

An entry is a directory built around one file, <name>.sse.tsv: a plain TSV with an extra Type row under the header that labels each column as an id, a label, or a plot coordinate. Initialization writes the rows once. After that, every stage below only adds columns through one shared merge_columns() contract — the amber cells are what a later tool appended.

Why an additive, append-only contract?

  • Rows are never reordered, removed, or multiplied
  • Joins are one-to-one on unique IDs, left-sided
  • Reserved columns (id, Sequence, query) can't be overwritten
  • Name collisions abort unless a tool passes force=True
  • Writes go to a temp file, then move atomically into place
  • A manifest records the tool, params, and version per column

The payoff: you can run stages in almost any order, rerun one without disturbing the others, and always trace where a column came from. The one rule — don't run two data-writing commands on the same entry at the same time.

The eight stages

From sequence file to foldable landscape

1

Create the entry

sse_initialization.py

Read a raw dataset, resolve one unique ID per sequence, validate the amino-acid content, and lay down the typed table plus a provenance manifest. This is bootstrap-only — it runs once.

InTSV · FASTA · Foldseek JSON
→
Doresolve IDs · validate · compute features
→
Out<name>.sse.tsv + manifest

Why here: everything downstream assumes unique, valid IDs and clean sequences. Catching duplicates and unusable sequences now — and aborting rather than half-writing — means no later tool has to defend against a broken table. Reference sequences are flagged as query so distance and structure tools can find them.

Choice · which source reader

emEnzymeMiner / TSV
Any tab-separated table with named ID + sequence columns. Keeps all extra columns as labels; auto-flags queries from a Closest query column.
fasta
When header-derived identity is enough. Resolves accessions from sp|…|…-style headers; rest of the header becomes Description. No automatic query.
fsFoldseek JSON
Unlocks structure-aware embeddings. Deduplicates targets across databases and carries C-alpha coordinates + taxonomy needed by ProstT5/SaProt and Foldseek taxonomy.

Also computed for every valid sequence: length, acidic/basic composition and ratios, and — for standard-residue sequences — net charge, MW, pI, aromaticity, instability index, and GRAVY. These come free and feed later cluster enrichment.

2

Annotate with taxonomy

fetch_taxonomy.py

Look up each sequence's organism and full lineage from NCBI, adding rank columns from superkingdom down to species.

Inrow IDs / taxIds
→
Doresolve taxId → NCBI lineage efetch
→
Outsuperkingdom … species

Why before clustering: taxonomy is one of the most useful things to test a cluster against — "is this group phylogenetically distinct, or just an artifact?" Adding it early means the Tier-2 analysis in stage 6 can run enrichment tests on it. A tax_status column marks each row as resolved, retryable, or definitively unavailable.

Choice · resolution strategy (auto-selected)

emaccession lookup
General case. Sends each ID to NCBI protein efetch. Works for any entry whose IDs are NCBI protein accessions.
foldseekdatabase-aware
For Foldseek entries. Uses existing taxIds, resolves GMGC unigene IDs, and records known no-taxonomy sources instead of guessing.

Reruns are cheap: --retry-failed only re-hits transient failures; --force refetches everything. Completed rows seed the next run so you never pay twice.

3

Merge external measurements

merge_external.py

Fold in your own lab data — activities, melting temperatures, any per-sequence measurement — matched by ID.

InCSV / TSV of measurements
→
Doexact ID join (optional translator)
→
Outlabel or coordinate columns

Why exact matching only: the merge deliberately refuses to guess identity. Trimming whitespace or stripping version suffixes automatically could silently mis-assign a measurement to the wrong protein — a far worse error than an empty cell. If IDs differ between systems, you supply a two-column translator so the mapping is explicit and auditable.

Choice · column type

--type labeldefault
For most data. Becomes filterable and can color points, but is not offered as a plot axis.
--type coordinate
For genuine numeric axes (e.g. a measured 2D assay). Makes the column selectable as an X or Y axis in the visualizer.
4

Embed & reduce

sse_coordinates.py

The heart of the pipeline: turn each sequence into a high-dimensional vector with a protein language model, then project those vectors down to plottable coordinates.

InSequence (text)
→
Embedmodel → per-residue → mean pool
→
ReducePCA / UMAP / t-SNE
→
Outesmc600m_mean_PC1…

Why two steps: the embedding places biologically similar sequences near each other in a ~1000-dimensional space — but you can't plot 1000 dimensions. Reduction compresses that geometry into 2–10 axes you can see and cluster. Why normalize (default): dividing each vector by its L2 norm fixes one shared geometry, so the coordinates you look at, the distances in stage 5, and the clusters in stage 6 all agree. The normalized cache is written once and reused everywhere.

Choice · embedder — what "similar" means

esmcstart here
Sequence-only similarity. Works on every entry; the general-purpose default for a sequence-space landscape.
prostt5Foldseek only
Structure-dominant. Embeds the 3Di structural alphabet, so proteins group by fold even when sequences have diverged.
saprotFoldseek only
Both at once. Combines amino-acid and structural tokens — useful when sequence and fold should each contribute.

Choice · reducer — what the map preserves

pcadeterministic
Global & linear. Same result every run, reports explained variance, and denoises the space for clustering. The dependable basis.
umap
The convenient 2D landscape. Nonlinear; preserves neighborhoods well and keeps some global structure. Usually the nicest to explore.
tsne
Local detail. Sharpens fine neighborhoods — but treat distances between clusters as unreliable; don't over-read the gaps.

Embeddings stream to a resumable cache, so a swap of reducer (or an interrupted run) never recomputes the model. The docs' advice: compare coordinate systems rather than assume one representation is universally best.

5

Distance to reference sequences

sse_esmc_distance.py

Measure how far every sequence sits from your query/reference sequences, in the full embedding space.

Inembedding cache + query rows
→
DoEuclidean distance to each query
→
Out…_distance_to_<query>

Why full-space, not the 2D map: the reduced coordinates distort distances for the sake of a nice picture. For "how close is this candidate to my known enzyme?", the honest answer comes from the complete embedding, so distance is computed there. The query's own distance is exactly zero — a built-in sanity check. These columns are ideal for coloring the map by similarity to a reference.

Defaults to every query=True row; pick specific references with --query-id. It reuses the same normalized cache, so distances share the geometry of the coordinates and clusters.

6

Cluster & profile

sse_cluster.py

Discover groups in the embedding space, mark a representative sequence per group, then automatically profile what makes each group distinct.

Inembedding cache
→
DoPCA denoise → cluster → analyze
→
Out_cluster · _representative · _dist_to_center + profiles

Why cluster the embedding, not the 2D plot: the plot is a lossy shadow. Clustering runs on the fuller (PCA-denoised) vectors so groups reflect real structure, not projection artifacts. Why a "representative": the real sequence nearest each center (a medoid) gives you an actual protein to inspect or fold — not a synthetic average that exists nowhere.

Choice · clustering algorithm

kmeanspartitions all
Every sequence gets a cluster. Sweeps k=2..20 and keeps the best silhouette. Best when compact, roughly round groups are a fair model.
hdbscandiscovers count
Irregular shapes + noise. Finds the number of clusters itself and leaves outliers as noise instead of forcing them into a group.

Choice · clustering space

--space pcadefault
Denoises first. Reduces to 50 PCs (or a variance target) — steadier k-means and much healthier HDBSCAN in wide embeddings.
--space full
No pre-reduction. Clusters the raw vectors directly when you specifically want the untouched geometry.

Tier-2 analysis (auto): after clustering, the tool regenerates three files per run so they can't drift from the labels — a profile (size, silhouette, median features, dominant categories), an enrichment table (Mann-Whitney U for numeric features, Fisher exact for categories, Benjamini-Hochberg FDR-corrected), and the top representatives as TSV + FASTA. This is why stages 2–3 come first: any annotation present becomes a testable hypothesis about each cluster.

Run both algorithms when cluster stability is itself the question. If Boltz scores arrive later and should be tested, rerun clustering with --force to refresh the analysis against the expanded table.

7

Predict structure & compare

sse_boltz.py

For a hand-picked selection, predict 3D structures with Boltz-2 and optionally measure how much each one deviates from a reference fold.

Inexported selection (from stage 8)
→
DoBoltz-2 fold → CIF · then RMSD align
→
OutpTM · pLDDT · RMSD_vs_<ref> columns

Why selection-driven: structure prediction is expensive, so it never runs on the whole dataset. You explore first (stage 8), select the handful worth folding, and only those are predicted. Why only scalars return to the table: the heavy CIF files live on disk; just the analysis-relevant confidence scores (and RMSDs) become columns you can filter and color by. Jobs are cached by identity, so reruns reuse completed folds.

Choice · what to fold, and how to compare

apono ligand
The protein alone. Leave SMILES empty. Yields boltz_apo_ptm/plddt and the structures RMSD can use.
holowith substrate
Protein + ligand complex. Supply one or more SMILES to model binding; scores are namespaced per ligand.
rmsd · seqdefault
Sequence-guided. Aligns by residue identity, rejects outliers, Kabsch-superposes Cα atoms. Good when sequences are alignable.
rmsd · ce
Structure-guided. CE alignment finds a superposition from geometry alone — for distant homologs. Run both to compare.

Requires an NVIDIA API key (BOLTZ_API_KEY). MSAs are generated and cached by default for accuracy; RMSD only scans completed apo structures and needs at least two.

8

Explore, select & export

sse_visualizer.py

The interactive payoff: a Dash app that plots the whole table, lets you filter and color by any column, overlay clusters, and lasso sequences worth folding.

Inthe full .sse.tsv
→
Doplot · filter · color · lasso · overlay clusters
→
Outlayers · figures · selection JSON

Why the table already knows everything: because each stage appended honest columns, the visualizer needs no special hooks — it reads the Type row, offers every coordinate system as axes, classifies every label for filtering, and turns any _cluster column into a region overlay. The loop back to stage 7: a lasso selection is exported to selections/…json, which the Boltz module then folds — closing the explore → select → predict cycle.

Selections become reusable colored layers (saved to disk), publication figures export as PNG/SVG/PDF, and visible layers extract to CSV. Reload after any new stage to surface its fresh columns.

Why it's built this way

Four ideas that make the whole thing hang together

Append-only

Order is flexible, provenance is not

Because tools only add columns and the manifest logs how each was made, you can rerun any stage in isolation and always reconstruct exactly which model, normalization, and parameters produced a result.

One geometry

Coordinates, distances, and clusters agree

Default L2 normalization writes a single shared embedding cache. The map you look at, the distances you measure, and the clusters you find are all computed in the same space — so they never contradict each other.

Annotate early

Metadata becomes a testable hypothesis

Taxonomy and lab measurements added before clustering get automatically enrichment-tested against every group, turning "these look different" into a statistic with an FDR-corrected p-value.

Explore, then spend

Expensive steps are selection-driven

Structure prediction never runs blindly on everything. You survey the cheap landscape first, then fold only the handful of sequences the exploration flagged as worth the compute.
